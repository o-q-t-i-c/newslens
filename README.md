# NewsLens v2(プロトタイプ)

同じニュースを「既成メディア」と「独立系Webメディア」がどう伝えているか——あるいは**どちらか一方しか伝えていないか**——をフィード形式で自動表示するサービス。

## 運営コスト:完全ゼロの設計

| 部品 | 使うもの | 料金 |
|---|---|---|
| 記事収集の自動実行(30分ごと) | GitHub Actions | 無料(Publicリポジトリは無制限) |
| サイト公開 | GitHub Pages | 無料(帯域100GB/月) |
| サーバー・データベース | 使わない | 0円 |
| AI・外部API | 使わない | 0円 |

サーバーが存在しないので、落ちる・攻撃される・請求が来る、がありません。

## 機能

- **自動収集**:11媒体(既成6+独立系5)のRSSから直近48時間の記事を取得
- **自動クラスタリング**:見出しの文字類似度で同一トピックをグルーピング(閾値は実測データで調整済み)
- **ブラインドスポット検出**:「両サイド比較」「独立系のみが報道」「既成のみが報道」をタブで切り替え
- **記事の蓄積**:RSSから消えた記事も48時間はデータに保持(前回結果と自動マージ)
- **報道バランスバー**:各トピックの既成/独立系の報道量を視覚化
- **PWA対応**:スマホのホーム画面に追加してアプリのように使える

## 公開手順(初回のみ・約15分)

1. [github.com](https://github.com) でアカウント作成(無料)
2. 「New repository」→ 名前 `newslens`、**Public** で作成
3. このフォルダの中身をすべてアップロード
   (「Add file → Upload files」でドラッグ&ドロップ。`.github` フォルダも忘れずに)
4. **Settings → Pages** → Source「Deploy from a branch」、Branch `main`、フォルダ `/docs` → Save
5. **Actions** タブ → 「ニュース自動更新」→「Run workflow」で初回実行
6. 数分後 `https://ユーザー名.github.io/newslens/` で公開

以降は30分ごとに全自動で更新され続けます。

## ローカル動作確認(任意)

Node.js 18以上があれば:

```bash
node scripts/fetch.mjs            # 実際のRSSから収集
node scripts/fetch.mjs --sample   # sample-feeds/ のテストデータで実行
npx serve docs                    # http://localhost:3000 で表示確認
```

## 調整ポイント

- `scripts/fetch.mjs` の `SOURCES`:媒体の追加・削除。RSSがある媒体なら1行で追加できる
- `SIMILARITY_THRESHOLD`(0.20):グルーピングの厳しさ。実測ではMERGE最小0.13 / SPLIT最大0.07なので0.20は安全マージン内
- `MAX_AGE_HOURS`(48):フィードに残す記事の期間

## 設計上の判断メモ

- **本文は転載しない**:見出し+リンクのみ。著作権リスクを避けつつ、クリックは各媒体に送る(媒体側との関係構築にも有利)
- **IDF重み付けは不採用**:実験の結果、ニュースの同一トピック内ではトピック語が繰り返されるためIDFが逆効果と判明。素のoverlap係数を採用
- **媒体の既成/独立系分類は手動定義**:ここはサービスの編集方針そのものなので、自動化すべきでない部分

## 次のステップ候補

1. 実運用でクラスタリング精度を観察し、閾値・記号除去ルールを調整
2. 独立系媒体の拡充(RSSを持つ媒体の調査)
3. 精度が頭打ちになったら:埋め込みAIによるクラスタリング(ここで初めてコストが発生。月数百円程度から)

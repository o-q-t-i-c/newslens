# NewsLens v3(プロトタイプ)

同じニュースを各媒体がどう伝えたか——そして**どの媒体が伝えていないか**——を自動で可視化するサービス。

## v3の考え方

左右のような政治的二分法や「既成 vs 独立系」の厳密な線引きは採用しない。日本の報道機関はそのように明確には分かれていないため。代わりに:

- **媒体カバレッジ表**:トピックごとに全媒体のアイコンを並べ、報じた媒体はカラー、フィードで確認できない媒体はグレー表示。「報道の空白」を事実ベースで見せる
- 媒体のグループ分けは「新聞・通信・放送」「ウェブメディア」という記述的な分類のみ
- グレー=「未確認」であり「報道していない」とは断定しない(RSSは各社の記事の一部しか含まないため)。この注記をフッターに常時表示

## 機能

- **自動収集**:11媒体・16フィードから直近48時間の記事を取得(30分ごと)
- **自動クラスタリング**:見出しの文字類似度で同一トピックをグルーピング
- **カテゴリタブ**:政治・経済・国際・社会・IT科学・スポーツ。カテゴリ別RSSがある媒体(NHK)はそれを利用、無い媒体はキーワードで自動分類
- **絞り込み**:「見比べられる(2媒体以上)」「単独報道」
- **サムネイル画像**:RSSに画像が含まれる場合のみ表示(記事ページからの取得はしない)
- **媒体アイコン**:Googleのファビコン取得サービスを利用(無料・ロゴの直接利用を回避)
- **続報の折りたたみ**:同一媒体の複数記事は最新1本+「他N件」にまとめる
- **記事の蓄積**:前回結果と自動マージし、RSSから消えた記事も48時間保持
- **PWA対応**:スマホのホーム画面に追加可能

## 公開手順(新規の場合)

1. GitHubで Public リポジトリ `newslens` を作成
2. `docs` `scripts` `sample-feeds` フォルダと `README.md` を**フォルダごと**アップロード
3. `.github/workflows/update.yml` は「Add file → Create new file」でGitHub上で直接作成(隠しフォルダ問題を回避)
4. Settings → Pages → Branch `main` / `/docs` → Save
5. Actions → 「ニュース自動更新」→ Run workflow

## 既存リポジトリの更新手順(v2からの差し替え)

1. リポジトリの「Add file → Upload files」で、新しい `scripts/fetch.mjs` と `docs/index.html` を**同じフォルダ構造で**アップロード(`scripts` `docs` フォルダごとドラッグすれば同名ファイルは上書きされる)
2. `README.md` も同様に上書き
3. Actions → Run workflow で手動実行すると即座に新形式のデータが生成される
   (旧 `data.json` は形式が違うが、スクリプトが自動で読み捨てて新形式で作り直す)

## 調整ポイント

- `scripts/fetch.mjs` の `OUTLETS` / `FEEDS`:媒体・フィードの追加削除
- `CATEGORY_RULES`:カテゴリ分類のキーワード辞書
- `SIMILARITY_THRESHOLD`(0.20):グルーピングの厳しさ(実測: MERGE最小0.13 / SPLIT最大0.07)

## 設計上の判断メモ

- 本文・画像は転載せず、RSSが提供する範囲のみ使用。クリックは各媒体に送る
- IDF重み付けクラスタリングは実験の結果、不採用(同一事件内でトピック語が繰り返されるため逆効果)
- 既知の限界:続報(例:利上げ決定→総裁会見)は見出しの重なりが薄いと別トピックになることがある。実データで頻度を観察して判断する
